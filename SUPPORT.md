# Support

## Getting help

- **Bug reports and feature requests** — open an issue in the relevant repository using the provided issue templates.
- **Contributing questions** — see [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines on branches, commits, and pull requests.
- **Security vulnerabilities** — see [SECURITY.md](SECURITY.md). Do not open public issues for security reports.
- **Org governance and automation** — questions about stale PR policy, label sync, repo settings, or Dependabot configuration are handled in [INORSA/org-management](https://github.com/INORSA/org-management).
- **Internal resources** — for Slack channels, internal documentation, and team contacts, check with your team lead.
